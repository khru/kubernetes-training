## Ruta recomendada según objetivo

Para **empezar bien**, usaría este orden:

1. **Kubernetes Docs + Learn Kubernetes Basics**
2. **Introduction to Kubernetes de Linux Foundation / edX**
3. **Play with Kubernetes o Killercoda** para practicar sin instalar nada
4. **Kubernetes in Action, Second Edition** como libro principal
5. **KodeKloud** si quieres labs guiados y preparación seria
6. **CKAD** si eres developer
7. **CKA** si quieres administración, plataforma, SRE o DevOps
8. **CKS** solo después de tener buena base y haber aprobado CKA
Para **developer**, priorizaría:

- Kubernetes in Action
- Kubernetes Patterns
- CKAD
- KodeKloud CKAD
- Learnk8s Blog para internals
Para **DevOps / SRE / Platform Engineer**, priorizaría:

- Kubernetes: Up and Running
- Kubernetes Best Practices
- Cloud Native DevOps with Kubernetes
- CKA
- Killer.sh
- Learnk8s workshops si tienes presupuesto
Para **seguridad**, priorizaría:

- CKA primero
- CKS después
- Kubernetes Best Practices
- Documentación oficial
- Labs de Killercoda / Killer.sh
Una nota importante: evitaría PDFs sueltos de libros comerciales que aparecen en Google. Para libros de O’Reilly, Manning u otras editoriales, mejor ir a la editorial, Amazon, VitalSource, O’Reilly Learning o bibliotecas legales. Kubernetes cambia bastante y además muchos PDFs no oficiales pueden estar desactualizados o no ser legales.
